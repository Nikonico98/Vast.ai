# Setup — Installation & Service Scripts
# 安装 — 安装与服务脚本

> **EN:** This folder contains scripts for **first-time setup**, **service management** (start/stop), and **migration** between GPU instances. You only need to run `setup.py` once when creating a new Vast.ai instance.
>
> **中文：** 这个文件夹包含**首次安装**、**服务管理**（启动/停止）和**GPU 实例迁移**的脚本。创建新的 Vast.ai 实例时，只需执行一次 `setup.py`。

---

## File Guide / 文件说明

| File / 文件 | When to use / 什么时候用 | Description / 说明 |
|:------------|:------------------------|:-------------------|
| `setup.py` | 🏗️ **First time only** / 仅首次 | The "renovation crew" — automatically installs everything: creates `sam3` and `sam3d-objects` conda environments, installs all Python packages, downloads SAM3 and SAM3D AI model weights from HuggingFace (~5-10 GB), installs trimesh for 3D processing. Takes 15-30 minutes on first run. With `--from-frozen` flag, can restore from exported snapshots (much faster). / "装修队"：自动安装一切。使用 `--from-frozen` 可从导出快照恢复（更快）。 |
| `export_envs.sh` | 📦 **Before migration** / 迁移前 | Exports current conda environments to frozen yml/pip files for migration. **Run this on the OLD instance before destroying it!** / 导出当前 conda 环境。**在销毁旧实例之前运行！** |
| `start.sh` | ▶️ **Every time** / 每次启动 | Starts GPU Service (gpu_app.py) + SAM3 persistent model servers (dual-GPU). Run after machine boots. / 启动 GPU 服务 + SAM3 持久化模型服务器（双 GPU）。 |
| `stop.sh` | ⏹️ **When stopping** / 停止时 | Gracefully stops all services: GPU Service + SAM3 model servers. / 优雅停止所有服务：GPU 服务 + SAM3 模型服务器。 |
| `install_trimesh.sh` | 🔧 **If trimesh is broken** / trimesh 出问题时 | Standalone script to install/reinstall `trimesh` in the SAM3D conda environment. / 独立脚本，在 SAM3D 环境中重装 trimesh。 |
| `envs/` | 📋 **Auto-generated** / 自动生成 | Frozen conda environment exports for migration. Generated by `export_envs.sh` or `setup.py`. / 导出的 conda 环境快照，用于迁移。 |

## Architecture / 架构

```
Dual-GPU Architecture (2x RTX A5000):
  GPU 0: SAM3 persistent server (port 5561) → real/photo items
  GPU 1: SAM3 persistent server (port 5571) → fictional items
  SAM3D: on-demand subprocess per job (saves VRAM)

Services:
  GPU Service (gpu_app.py) → port 5555 (main API)
  SAM3@GPU0 (sam3_server.py) → port 5561
  SAM3@GPU1 (sam3_server.py) → port 5571
```

## Migration to a New Instance / 迁移到新实例

### Step 1: On the OLD instance (before destroying) / 在旧实例上

```bash
# Export environment snapshots / 导出环境快照
cd /workspace/IW/setup
bash export_envs.sh

# Commit to git / 提交到 git
cd /workspace/IW
git add setup/envs/
git commit -m "Export frozen envs for migration"
git push
```

### Step 2: On the NEW instance / 在新实例上

```bash
# 1. Clone / pull the repo / 克隆 / 拉取代码
cd /workspace
git clone <your-repo-url> IW   # or: cd IW && git pull

# 2. Run setup with --from-frozen (faster restore) / 使用 --from-frozen 安装
cd /workspace/IW/setup
python setup.py --from-frozen

# 3. Configure .env / 配置环境变量
cd /workspace/IW/backend
cp .env.example .env
nano .env    # Fill in GPU_API_SECRET, HF_TOKEN, etc.

# 4. Start services / 启动服务
cd /workspace/IW/setup
bash start.sh
```

### Step 3: Update Hostinger / 更新 Hostinger 配置

Update `GPU_WORKER_URL` in Hostinger's `.env` with the new Vast.ai instance IP.

## What setup.py does (in detail) / setup.py 做了什么（详细版）

1. **Creates conda environment `sam3`** — Python 3.12 + SAM3 model dependencies
   创建 conda 环境 `sam3` — Python 3.12 + SAM3 模型依赖

2. **Creates conda environment `sam3d-objects`** — Python + SAM3D model dependencies
   创建 conda 环境 `sam3d-objects` — Python + SAM3D 模型依赖

3. **Downloads AI model weights** — From HuggingFace (needs `HF_TOKEN`), several GB each
   下载 AI 模型权重 — 从 HuggingFace 下载（需要 `HF_TOKEN`）

4. **Installs trimesh** — 3D model processing library for GLB post-processing
   安装 trimesh — GLB 后处理用的 3D 模型处理库

5. **Verifies GPU access** — Checks NVIDIA drivers and CUDA
   验证 GPU 访问 — 检查 NVIDIA 驱动和 CUDA

6. **Exports frozen envs** — Saves snapshots for future migration
   导出环境快照 — 保存快照以备将来迁移

## Troubleshooting / 故障排除

**Q: setup.py fails downloading models / setup.py 下载模型失败**
A: Check your `HF_TOKEN` is valid. You may need to accept the model license on HuggingFace first.

**Q: `--from-frozen` fails / 从快照恢复失败**
A: The frozen yml may have platform-specific packages. Try full install: `python setup.py` (without `--from-frozen`).

**Q: "conda not found" error / 报错 "conda not found"**
A: Make sure conda/miniconda is installed. On Vast.ai, usually at `/opt/conda` or `/opt/miniforge3`.

**Q: trimesh import error / trimesh 导入错误**
A: Run `bash install_trimesh.sh` to reinstall it in the correct conda environment.

**Q: GPU not detected / 检测不到 GPU**
A: Run `nvidia-smi` to check NVIDIA drivers. Make sure you selected a GPU instance on Vast.ai.

**Q: SAM3 model servers won't start / SAM3 模型服务器启动失败**
A: Check logs: `tail -f /workspace/IW/logs/sam3_gpu0_server.log`
